# 项目管理页面提示词

本文档用于给 Google AI Studio 生成项目管理平台页面设计稿。

范围说明：
- 需要：除“排期协同”以外的管理页面、填报页面
- 不需要：排期协同页面
- 当前不纳入：分析中心甘特 BI 展示页

建议使用方式：
1. 先复制“统一风格基线提示词”
2. 再追加某一个具体页面提示词
3. 让 Google AI Studio 输出高保真 Web 页面设计稿

---

## 统一风格基线提示词

```text
请为一个企业级“项目管理平台”设计高保真 Web 管理后台页面，语言为简体中文。

设计要求如下：
1. 页面风格需要与现有设计平台保持一致，属于同一产品体系，不能是另一套完全不同的风格。
2. 整体采用企业级、专业、清爽、浅色、现代的管理后台风格，强调秩序感、信息密度和可操作性。
3. 页面结构统一为：
   - 顶部：系统级头部区域，可包含面包屑、全局搜索、通知、用户信息、退出按钮
   - 左侧：菜单导航区域
   - 中间：主内容区，支持 tab 卡片切换或标准内容布局
4. 视觉关键词：
   - 浅蓝、白色、淡灰为主色
   - 卡片圆角柔和，但不能过于花哨
   - 分区清晰，表格、表单、筛选区、统计卡片层次明确
   - 字体清晰、专业，强调表格和表单易读性
5. 页面必须适合真实后台系统，不要做成营销官网，不要做成过度炫技的大屏。
6. 页面内容以“管理、录入、查询、审核、维护”为主，强调可用性。
7. 所有文案使用中文，字段名、按钮名、标签名、表头名都要贴近中国企业项目管理场景。
8. 需要体现真实业务感，不能只放空卡片和假模块名。
9. 每个页面都要考虑这些状态：
   - 正常数据态
   - 空态
   - 新增或编辑弹窗 / 抽屉
   - 表格筛选区
10. 页面输出重点是 UI 设计，不需要写后端代码，不需要数据库设计。

请输出：
- 一张完整页面设计说明
- 页面结构分区说明
- 核心模块说明
- 关键表格字段 / 表单字段建议
- 关键按钮和交互建议

页面要体现“这是一个真实可落地的企业项目管理平台页面”。
```

## 页面 1：项目台账

```text
请基于统一风格基线，为“项目管理平台”的【项目台账】页面设计高保真后台页面。

页面定位：
- 这是项目管理的主台账页面
- 用于查看、筛选、创建、编辑、删除项目
- 这是管理页，不是甘特图页

页面目标：
- 管理员快速查看所有项目
- 支持新增项目、编辑项目、删除项目、按项目类型初始化
- 支持从列表进入某个项目的后续管理页面

页面内容要求：
1. 顶部需要有页面标题、页面说明、刷新按钮、新增项目按钮
2. 页面上方需要有筛选区，至少包含：
   - 项目编码搜索
   - 项目名称搜索
   - 项目状态筛选
   - 项目类型筛选
   - 项目经理筛选
   - 日期范围筛选
3. 中间主体为项目列表表格
4. 表格字段建议包含：
   - 项目编码
   - 项目名称
   - 项目类型
   - 项目经理
   - 项目状态
   - 来源说明
   - 计划开始日期
   - 计划结束日期
   - 项目预算
   - 团队人数
   - 创建时间
   - 操作列
5. 操作列建议包含：
   - 查看
   - 编辑
   - 初始化节点任务
   - 删除
6. 页面右侧或顶部可以有统计卡片：
   - 项目总数
   - 进行中项目数
   - 已完成项目数
   - 暂停项目数
7. 需要设计“新增 / 编辑项目”弹窗或抽屉
8. 新增 / 编辑表单建议字段：
   - 项目编码
   - 项目名称
   - 项目类型
   - 项目经理
   - 项目描述
   - 业务单元
   - 来源说明
   - 计划开始日期
   - 计划结束日期
   - 考勤地址
   - 预算金额
   - 状态
   - 备注
9. 风格上要突出“项目总台账”的管理感，表格为主，表单辅助

请输出一个真实可落地的企业后台【项目台账】页面设计稿说明。
```

## 页面 2：里程碑模板管理

```text
请基于统一风格基线，为“项目管理平台”的【里程碑模板管理】页面设计高保真后台页面。

业务背景：
- 里程碑模板管理 = 项目类型 + 节点模板 + 任务模板 + 模板排期
- 它是项目执行前的基础配置页面
- 它不是项目实例页面，而是模板配置页面

页面目标：
- 管理员维护不同项目类型的标准模板
- 配置每个项目类型下的节点模板
- 配置每个节点模板下的任务模板
- 配置模板排期和基础规则

页面结构建议：
1. 页面上方：
   - 标题：里程碑模板管理
   - 页面说明
   - 新增模板按钮
   - 导入 / 导出 / 复制模板入口可以预留
2. 主体建议为三栏结构或左中右结构：
   - 左栏：项目类型 / 模板列表
   - 中栏：节点模板列表
   - 右栏：任务模板列表与模板属性
3. 也可以设计为“上模板列表 + 下节点任务配置”的后台布局，但必须清晰表达三层关系

页面内容要求：
1. 模板头字段建议：
   - 模板编码
   - 模板名称
   - 项目类型名称
   - 模板说明
   - 状态
   - 排序
2. 节点模板字段建议：
   - 节点编码
   - 节点名称
   - 上级节点
   - 层级
   - 排序
   - 是否需验收
   - 是否需审核
   - 备注
3. 任务模板字段建议：
   - 任务编码
   - 任务名称
   - 所属节点
   - 任务说明
   - 是否需验收
   - 是否需审核
   - 是否需上传资料
   - 是否需结算
   - 备注
4. 模板排期区域建议体现：
   - 按月、日维度配置计划区间
   - 可显示每个任务模板的计划时间块
   - 这里只做管理页风格，不需要做排期协同的完整图形界面
5. 操作建议包含：
   - 新增模板
   - 复制模板
   - 编辑模板
   - 删除模板
   - 新增节点模板
   - 新增任务模板
   - 批量维护排序

交互要求：
- 明确区分“模板头”、“节点模板”、“任务模板”
- 适合频繁配置，不要做得太花
- 强调结构清晰和编辑效率

请输出一个适合企业后台使用的【里程碑模板管理】页面设计稿说明。
```

## 页面 3：任务填报

```text
请基于统一风格基线，为“项目管理平台”的【任务填报】页面设计高保真后台页面。

业务定位：
- 这是成员填报页面
- 用于任务接收、进度反馈、完成说明提交
- 这是独立页面，不放到甘特图管理页里

页面目标：
- 项目成员查看自己负责或参与的任务
- 对任务进行接收、更新进度、填写完成说明
- 管理员可查看填报状态

页面内容要求：
1. 顶部区域需要有：
   - 页面标题
   - 我的待处理任务数
   - 我的进行中任务数
   - 我的已完成任务数
2. 筛选区建议包含：
   - 项目名称
   - 任务状态
   - 节点名称
   - 责任人 / 当前用户
   - 日期范围
3. 主体建议采用：
   - 上方统计卡片
   - 下方任务列表表格
   - 右侧或弹窗任务填报表单
4. 任务表格字段建议：
   - 项目名称
   - 节点名称
   - 任务名称
   - 责任人
   - 参与人员
   - 计划开始时间
   - 计划结束时间
   - 当前状态
   - 验收状态
   - 审核状态
   - 当前进度
   - 操作
5. 任务填报弹窗 / 抽屉字段建议：
   - 是否接收任务
   - 当前进度百分比
   - 完成说明
   - 协调事项
   - 风险说明
   - 附件上传入口
   - 提交按钮
6. 操作按钮建议：
   - 接收任务
   - 开始处理
   - 保存草稿
   - 提交完成
   - 查看历史填报

风格要求：
- 这是“成员执行页”，要比项目台账更偏操作和填报
- 保持后台感，但增加一定的任务状态识别感
- 重点突出“当前待办”和“快速填报”

请输出一个真实可落地的【任务填报】页面设计稿说明。
```

## 页面 4：计划与日志

```text
请基于统一风格基线，为“项目管理平台”的【计划与日志】页面设计高保真后台页面。

业务定位：
- 这是独立的填报与管理页面
- 用于项目的月计划、周计划、日计划、总结、日志录入
- 计划由项目经理制定，日志与总结由成员或管理者填报

页面目标：
- 管理项目计划从月到周、周到日的拆解
- 记录执行日志
- 记录周总结、月总结

页面内容要求：

2. 页面结构建议：

   - 中间主体：对应列表 + 编辑区
3. 计划列表字段建议：
   - 项目名称

   - 计划开始日期
   - 计划结束日期
   - 负责人

4. 日志 / 总结列表字段建议：
   - 日期
   - 填报人
   - 所属项目
   - 所属节点 / 任务
   - 日志类型
   - 填报状态
   - 操作
5. 编辑表单建议字段：
   - 计划类型
   - 所属项目
   - 所属节点 / 任务
   - 开始日期
   - 结束日期
   - 计划内容
   - 负责人
   - 状态
   - 备注
6. 日志 / 总结表单字段建议：
   - 填报日期
   - 所属项目
   - 所属节点 / 任务
   - 今日完成内容
   - 遇到问题
   - 协调事项
   - 明日计划
   - 总结内容
   - 备注

交互要求：
- 页面要体现“计划管理”和“日志记录”两种不同性质
- 计划更偏结构化和周期管理
- 日志更偏内容录入和时间线
- 可以使用 tab、分栏、抽屉组合，但必须易用

请输出一个适合企业后台使用的【计划与日志】页面设计稿说明。
```

```text
请按真实企业后台产品页面来设计，强调：
- 筛选区
- 表格区
- 统计区
- 弹窗 / 抽屉
- 操作按钮
- 中文业务字段

请尽量让页面一眼看上去就能用于“项目管理系统”的实际开发落地。
```
