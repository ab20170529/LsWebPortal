# Week 2 Execution Plan V1

This document starts week 2 work for the hybrid PM + UI takeover track.

Week 1 outcome:

- product structure is clear
- major role and UX risks are documented
- IA direction is proposed
- first key pages have low-fi specs

Week 2 goal:

move from analysis into design execution.

---

## 1. Week 2 Objective

This week should produce the first real design-ready page structures for the highest-value surfaces:

1. system gate page
2. project shell
3. project ledger

These three pages define:

- first understanding
- navigation logic
- project entry and control logic

If these three are stable, later pages become much easier to design.

---

## 2. Week 2 Deliverables

### Must Deliver

1. Figma-ready page block map for system gate
2. Figma-ready page block map for project shell
3. Figma-ready page block map for project ledger
4. UI copy direction for the three pages
5. open decision list for business/engineering review

### Good To Deliver

1. one alternate variant for system gate emphasis
2. one role-based variant for project shell landing
3. one "empty state / no data state" sheet for project ledger

---

## 3. This Week's Working Sequence

### Step 1

Freeze the structural model:

- context first, system second
- role-aware project workbench
- project ledger as the project control entry

### Step 2

Design system gate as a controlled launchpad.

### Step 3

Design project shell as the durable work frame.

### Step 4

Design project ledger as the operational entry surface.

### Step 5

Review with engineering and business before moving to polished visual refinement.

---

## 4. Recommended Day Split

### Day 1

- system gate block structure
- system gate content hierarchy
- system gate state variants

### Day 2

- project shell block structure
- navigation grouping
- role-based default entry logic

### Day 3

- project ledger block structure
- detail/list relationship
- initialization action treatment

### Day 4

- review and revise the three structures
- create open issue list
- align with engineering constraints

### Day 5

- freeze wireframe direction
- prepare high-fidelity start package

---

## 5. Priority Rules For Week 2

1. solve structure before style
2. solve hierarchy before decoration
3. solve role logic before screen polish
4. solve states before motion or visual flavor

---

## 6. Review Questions For Week 2

### System Gate

1. Is the company-selection dependency now obvious?
2. Does the user know what Project is for before entering?

### Project Shell

1. Does the shell reflect role intent?
2. Can users locate their next action fast?

### Project Ledger

1. Does the page support scanning and continuation of work?
2. Is initialization-by-template prominent enough?

---

## 7. Exit Criteria For Week 2

Week 2 is successful if:

1. the three target pages have stable wireframe structure
2. the open decisions are small enough to review quickly
3. visual design can begin without revisiting information architecture
