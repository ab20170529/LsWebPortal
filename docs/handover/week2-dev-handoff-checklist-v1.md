# Week 2 Dev Handoff Checklist V1

Use this checklist when handing the week-2 outputs to engineering.

This is not a ticket template.

It is a handoff quality checklist.

---

## 1. Handoff Package Must Include

- [ ] target page name
- [ ] business goal
- [ ] role impact
- [ ] Figma-ready structure
- [ ] copy sheet references
- [ ] state list
- [ ] open decisions if any remain

---

## 2. System Gate Handoff

Engineering should receive:

- [ ] page structure
- [ ] locked vs ready behavior
- [ ] company switching behavior
- [ ] no-company behavior
- [ ] load-error behavior
- [ ] final or provisional copy

Questions to confirm:

- [ ] is route logic unchanged?
- [ ] is current company activation flow unchanged?
- [ ] are admin entry conditions unchanged?

---

## 3. Project Shell Handoff

Engineering should receive:

- [ ] new nav grouping
- [ ] role-based default landing rule
- [ ] current project visibility rule
- [ ] no-project-selected guidance rule

Questions to confirm:

- [ ] does implementation require permission-selection updates?
- [ ] are tabs kept, reduced, or re-styled only?

---

## 4. Project Ledger Handoff

Engineering should receive:

- [ ] list/detail layout change
- [ ] selected row behavior
- [ ] initialization emphasis rule
- [ ] action hierarchy
- [ ] empty/error/no-permission states

Questions to confirm:

- [ ] are current APIs enough?
- [ ] does initialization status need new backend fields?
- [ ] are permission-controlled actions already enforced server-side?

---

## 5. Final Engineering Check

Before implementation starts:

- [ ] no major IA uncertainty remains
- [ ] role default behavior is confirmed
- [ ] copy is sufficiently stable
- [ ] state design is complete enough to implement without guessing
