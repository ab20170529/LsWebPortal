# 项目甘特工作区结构设计

## 目标

项目管理平台后续要把“新增项目”和“关联设置”逐步收拢到一个统一的甘特工作区里完成，但这个工作区不应和当前“模板配置”混在一起。

这份文档先明确第一版结构边界，保证后续设计图落地时不会把模板态和实例态再次耦合。

## 状态边界

### 模板配置态

入口：
- `进度配置`

职责：
- 维护项目类型
- 维护节点模板
- 维护任务模板
- 维护模板排期块

数据来源：
- `project type`
- `project type node`
- `project type task`
- `project type task schedule`

特点：
- 面向“模板”
- 不绑定某一个实际项目
- 配置结果用于后续项目初始化

### 项目实例态

入口：
- `项目管理`
- `项目甘特工作区`

职责：
- 新增项目实例
- 选择或切换项目类型
- 配置来源说明、业务单元、考勤地址等项目主档
- 管理项目成员、预算、附件、计划和实际执行
- 在甘特工作区里进行项目节点与任务的集中查看和后续编辑

数据来源：
- `project`
- `project member`
- `project budget`
- `project node`
- `project task`
- `plan/report/checkin/attachment`

特点：
- 面向“实际项目”
- 绑定当前选中的项目实例
- 是后续你要的“新增项目 + 关联设置 + 甘特编辑”的主战场

## 页面结构

项目平台主页维持：
- 顶部头部
- 左侧工作区菜单
- 中间 tab 内容区

中间内容区建议固定为 4 个工作区：

1. 项目管理
2. 项目总览
3. 进度配置
4. 项目甘特工作区

其中：
- `项目管理` 负责项目主档 CRUD 和初始化
- `项目总览` 负责执行态只读查看
- `进度配置` 负责模板配置
- `项目甘特工作区` 负责未来实例态的一体化操作

## 项目甘特工作区建议分区

### 顶部上下文区

内容：
- 当前项目
- 项目类型
- 项目经理
- 项目周期
- 当前状态

作用：
- 让用户明确当前正在编辑哪个项目实例

### 左侧准备区

内容：
- 项目基础信息
- 关联设置
- 成员与角色
- 预算与费用
- 资料与附件

作用：
- 作为甘特画布外的辅助配置入口

### 中央甘特主画布

内容：
- 月 / 日时间轴
- 节点泳道
- 任务条块
- 里程碑标记

作用：
- 承载项目实例的核心可视化编辑与查看

### 右侧详情区

内容：
- 当前选中节点或任务详情
- 时间
- 责任人
- 参与人
- 资料
- 审核 / 验收状态

作用：
- 让用户在不离开画布的情况下完成局部设置

## 实施顺序

### 第一步

在项目平台中先挂出 `项目甘特工作区` 骨架页面，只放结构，不提前固化交互细节。

### 第二步

把当前选中项目的上下文、基础统计和目标工作区说明接进去，作为可演示占位。

### 第三步

等设计图到位后，再逐步把下面能力并入：
- 新增项目
- 关联设置
- 成员维护
- 预算维护
- 实例节点任务编辑
- 甘特主画布交互

## 当前结论

现阶段不直接把“新增项目”硬塞进现有模板配置页，而是单独建立 `项目甘特工作区` 作为未来承接实例态的一体化入口。

这样可以保证：
- 模板配置和项目实例不串台
- 当前代码结构可继续演进
- 等设计图出来后不需要再推翻整个平台信息架构
