## bi界面设计稿
# bi界面包括两个部分，一个是设计界面，二个是bi展示界面，以下是设计平台的文案
-  这是一个设计平台，整个平台我希望以可视化操作的方式来构建。主要包含以下几大功能模块
-  1.整个目录结构管理 公司/大维度 -> 部门 -> 分析维度 -> 子维度，并允许后续继续扩展层级，我希望以画布的方式，如画流程的方式来构建，直观，清晰
-  2.分析源管理，分析源是指定这个节点所有数据可能来自于哪些表，或哪些sql，表对应的字段分别什么意思。
  - 每个目录节点可挂多个分析源，如果某个表的分析源已存在，可直接引用，如果没有，可一键根据表或sql生成字段说明，并读取表字段说明作为说明文案
-  3.BI 类型统一为 INTERNAL（系统自建）或 EXTERNAL（外链/iframe），需统一将bi数据作为档案管理起来（建立bi档案表）
